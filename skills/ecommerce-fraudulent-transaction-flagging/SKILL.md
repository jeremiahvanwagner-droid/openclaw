# Fraudulent Transaction Flagging

1. Ingest transaction, device, velocity, and behavioral risk signals.
2. Score fraud likelihood using rule and anomaly detection layers.
3. Classify risk tiers with explainable trigger reasons.
4. Apply automatic action by policy (approve, hold, block).
5. Alert operations on high-value or coordinated attack patterns.
6. Feed confirmed outcomes back into model calibration.
7. Output fraud dashboard with false-positive tracking.
