# Billing Reconciliation

1. Ingest payment events and subscription status changes.
2. Detect failed payments, dunning states, and access mismatches.
3. Trigger retry/dunning sequences based on policy windows.
4. Apply access revocation or restoration rules deterministically.
5. Log reconciliation actions for finance and support auditability.
6. Escalate edge cases requiring manual review.
7. Output daily reconciliation report with exception list.
